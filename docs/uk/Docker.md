Це конфігурація volumes у Docker Compose, яка використовується для керування тим, як файли та каталоги спільно
використовуються між вашою хост-машиною (вашим комп'ютером) та контейнером Docker.

Ось роз'яснення для кожного рядка:

1.  `- .:/app`
    - `.` (крапка): Це посилається на поточний робочий каталог на вашій хост-машині, де знаходиться ваш файл
      docker-compose.develop.yml. У вашому випадку це /Users/z1mak/projects/z1mak-cv-queue.
    - `/app`: Це посилається на каталог /app всередині контейнера Docker. У вашому Dockerfile встановлено WORKDIR
      /app, що робить /app робочим каталогом для вашої програми всередині контейнера.
    - `:` (двокрапка): Розділяє шлях на хості від шляху в контейнері.
    - Що це робить: Цей рядок монтує (приєднує) весь каталог вашого проєкту з хост-машини до каталогу /app
      всередині контейнера. Це означає, що будь-які зміни, які ви вносите у вихідний код на хост-машині, негайно
      відображаються всередині контейнера, без необхідності перезбірки образу Docker. Це важливо для live-reload і
      швидкого циклу розробки.

2.  `- /app/node_modules`
    - `/app/node_modules`: Це посилається на каталог node_modules всередині контейнера Docker. Це анонімний том
      (оскільки перед двокрапкою не вказано жодного шляху на хості).
    - Що це робить: Коли ви вказуєте анонімний том таким чином для каталогу, який вже існує в образі Docker (тому
      що npm ci встановив залежності в /app/node_modules під час етапу збірки), це, по суті, вказує Docker:
      "Використовуй каталог node_modules, який був створений під час збірки образу, і не перезаписуй його жодним
      каталогом node_modules, який може існувати на хост-машині в монтуванні .:/app."
    - Чому це потрібно: Без цього рядка, якщо у вас є каталог node_modules на вашій хост-машині (наприклад, якщо
      ви запускали npm install локально), монтування .:/app ефективно перезаписало б node_modules всередині
      контейнера (потенційно несумісними) node_modules з вашого хосту. Вказавши /app/node_modules як окремий том,
      ви гарантуєте, що node_modules, встановлені всередині контейнера під час кроку npm ci, залишаються
      ізольованими та використовуються контейнеризованою програмою, запобігаючи потенційним конфліктам залежностей
      між вашим хостом та середовищами контейнерів.

Підсумовуючи, ця конфігурація volumes забезпечує:

- Миттєві зміни коду: Редагуйте код на хості, одразу бачте зміни в контейнері.
- Ізольовані залежності: Забезпечує використання контейнером власного набору node_modules, сумісних з його версією
  Node.js та середовищем, уникаючи конфліктів з локальними node_modules вашого хоста.
